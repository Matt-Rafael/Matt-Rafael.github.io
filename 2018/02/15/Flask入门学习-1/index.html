Flask入门学习<1>

发表于 2018-02-15 | 分类于 Web开发

该篇文章主要是包含Flask的基本入门：安装、基本的应用结构、模板、表单

安装Flask

虚拟环境是Python解释器的一份拷贝，可以在里面私有地安装包。

$ python3 -m venv matt 创建名字为matt的虚拟环境
$ source matt/bin/activate 激活matt目录下的虚拟环境
$ deactivate 取消当前的虚拟环境
$ pip install flask 安装Flask
$ pip freeze 列出当前安装的包

应用结构

初始化

from flask import Flask
app = Flask(__name__)

所有的Flask应用必须创建应用实例，服务器将所有的请求都交给该对象处理。Flask使用__name__这个参数来决定应用的根路径。

路由和视图函数

通过app.route装饰器来定义路由
@app.route('/')
def index():
    return '<h1>Hello World<h1>'

通过app.add_url_rule()方法来定义路由
def index():
    return '<h1>Hello World<h1>'
app.add_url_rule('/', 'index', index)

使用<>来支持动态路由
@app.route('/user/<name>')
def user(name):
    return '<h1>Hello, {}'.format(name)

开发服务器

使用flask run命令开启服务器，该命令需要寻找包含应用实例的脚本
$ export FLASK_APP=main.py
$ flask run

通过调用app.run()方法来语法式地开启服务器：
if __name__ == '__main__':
    app.run()

调试模式

调试模式下，开发服务器的两个模块reloader和debugger默认开启。源文件改变的时候，reloader会自动重启服务器。当应用抛出异常时，debugger就会出现在浏览器中。

命令行开启调试模式：
$ export FLASK_APP=main.py
$ export FLASK_DEBUG=1
$ flask run

语法式开启调试模式：
app.run(debug=True)

请求-响应

应用和请求上下文

Flask接收到来自客户端的请求时，需要让处理请求的视图函数能够获取到一些对象，比如request对象。

Flask使用上下文来暂时让某些对象可全局获取：

from flask import request
@app.route('/')
def index():
        user_agent = request.headers.get('User_Agent')
        return '<p>Your browser is {} <p>'.format(user_agent)


    Flask的上下文全局变量：current_app、g、request、response

请求钩子

在请求被处理之前或之后执行代码，使用钩子来实现：
 before_first_request：第一个请求被处理之前执行
 before_request：每个请求被处理之前执行
 after_request：每个请求被处理之后执行，前提是没有异常发生
 teardown_request：每个请求被处理之后执行，就算异常发生

响应对象

返回tuple作为响应：
@app.route('/')
def index():
    return '<h1>Bad Request<h1>', 400

返回response对象：
from flask import make_response
@app.route('/')
def index():
    response = make_response('<h1>This document carries a cookie<h1>')
    response.set_cookie('answer', '42')
    return response


重定向：
from flask import redirect
@app.route('/')
def index():
    return redirect('https://www.baidu.com')

abort()函数用于错误处理：
from flask import abort
@app.route('/user/<id>')
def get_user(id):
    user = load_user(id)
    if not user:
        abort(404)
    return '<h1>Hello, {}<h1>'.format(user.name)

模板

默认Flask寻找位于应用主目录下名字是templates子目录的模板

templates/index.html：
<h1>Hello, World</h1>

templates/user.html：
<h1>Hello, {{ name }}<h1>

main.py：
from flask import Flask, render_template
app = Flask(__name__)

@app.route('/')
def index():
        return render_template('index.html')

@app.route('/user/<name>')
def user(name):
        return render_template('user.html', name=name)

    Jinja2识别任何类型的变量，甚至是列表，字典和对象。可以使用过滤器来修改变量：Hello, {{ name | capitalize }}

    Bootstra集成

    $ pip install flask-bootstrap

集成：
from flask_bootstrap import Bootstrap
bootstrap = Bootstrap(app)

Flask-Bootstrap初始化后，应用便可获取包含Boostrap文件和基本结构的模板，然后利用Jinja2的模板继承的特点。

    自定义错误页

    客户端请求的页面或路由不存在时，抛出404；应用有未处理的异常时，抛出500。
@app.errorhandler(404)
def  page_not_found(e):
    return render_template('404.html'), 404

@app.errorhandler(500)
def internal_server_error(e):
    return render_template('500.html'), 500

    链接

    url_for()函数产生URL，该函数接收视图函数名或者端点名。
url_for('index')返回相对路径
url_for('index', _external=True)返回绝对路径

动态URL可以通过传递参数来产生：
url_for('user', name='matt', _external=True)

查询字符串也可以作为参数传递：
url_for('user', name='matt', page=1, _external=True) page就是查询字符串

    静态文件

    静态文件默认放在应用根目录下名字叫static的子目录下。
url_for('static', filename='css/style.css', _external=True)

    Web表单

    $ pip install flask-wtf

Flask-WTF扩展不需要初始化，但是它需要应用配置secret key。
app = Flask(__name__)
app.config['SECRET_KEY'] = 'secret_string'

    表单类

    from flask_wtf import FlaskForm
from wtforms import StringField, SubmitField
from wtforms.validators import DataRequired

class NameForm(FlaskForm):
        name = StringField('What is your name?', validators=[DataRequired()])
        submit = SubmitField('Submit')

    @app.route('/', methods=['GET','POST'])
def index():
        name = None
       form = NameForm()
        if form.validate_on_submit():
                name = form.name.data
             form.name.data = ''
        return render_template('index.html', form=form, name=name)
